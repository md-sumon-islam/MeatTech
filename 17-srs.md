# Software Requirements Specification (SRS) - Core Overview

## 1. Project Reference Context
This compilation serves as the formal architectural requirements source index for **MeatTech**, a modern web application dedicated to managing the end-to-end supply chain, quality auditing, and B2C/B2B fulfillment of fresh meat distribution operations.

## 2. Intended Audience
This document is structured specifically to align development teams, quality assurance personnel, and system operators on technical scope boundaries before development begins.

## 3. Subdocument Cross-Reference Hierarchy
The complete technical requirements profile is split into modular segments for accessibility:
* Detailed feature sets can be audited via the [Functional Requirements Guide](./13-functional-requirements.md).
* Operational restrictions are outlined in the [Non-Functional Requirements Log](./14-non-functional-requirements.md).
* Flow expectations are visually represented inside the [System Use Case Library](./15-use-cases.md).
* Data transformations are traceable across the [Data Flow Diagram Matrix](./16-dfd.md).
